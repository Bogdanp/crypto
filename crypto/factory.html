2 Cryptography Providers

This library relies on foreign libraries for the implementations of cryptographic primitives. Each foreign library is called a cryptography provider and it has an associated factory that map cryptographic algorithm specifiers to implementations. A cryptography provider may also export other functions—for example, to initialize a random number generator.

Cryptography providers may be used to obtain algorithm implementations either explicitly or implicitly. An implementation of a cryptographic algorithm may be obtained explicitly by calling the appropriate function (eg, get-digest, get-cipher, etc) on a factory or list of factories. Alternatively, most functions provided by this library for performing cryptographic operations accept either a implementation object or an algorithm specifier. If an algorithm specifier is given, an implementation is implicitly sought from the factories in (crypto-factories); if no implementation is available, an exception is raised.

procedure

(crypto-factory? v) → boolean?

  v : any/c
Returns #t if v is a crypto factory, #f otherwise.

parameter

(crypto-factories) → (listof crypto-factory?)

(crypto-factories factories) → void?
  factories : (listof crypto-factory?)
The list of crypto factories used when implicitly finding an implementation of a cryptographic algorithm from an algorithm specifier.

The initial value is '().

procedure

(get-factory i) → crypto-factory?

  i : 
(or/c digest-impl? digest-ctx?
      cipher-impl? cipher-ctx?
      pk-impl? pk-parameters? pk-key?
      random-impl?)
Gets the factory associated with a particular cryptographic algorithm implementation.

2.1 Libcrypto (OpenSSL)

 (require crypto/libcrypto) package: crypto-lib

value

libcrypto-factory : crypto-factory?

Factory for libcrypto, the cryptography library of OpenSSL. The necessary foreign library is typically part of the operating system or distributed with Racket.

CSPRNG Initialization The libcrypto library automatically seeds its CSPRNG using entropy obtained from the operating system (as described here).

2.2 GCrypt

 (require crypto/gcrypt) package: crypto-lib

value

gcrypt-factory : crypto-factory?

Factory for GCrypt, a cryptography library from the GNU project, originally part of GnuPG. The libgcrypt.so.11 foreign library is required.

CSPRNG Initialization The libgcrypt library seems to perform some default CSPRNG initialization, but I don’t know the details.

2.3 Nettle

 (require crypto/nettle) package: crypto-lib

value

nettle-factory : crypto-factory?

Factory for Nettle, a lightweight cryptography library. The libnettle.so.4 foreign library is required.

CSPRNG Initialization This library creates a Yarrow-256 instance and seeds it with entropy obtained from the operating system via /dev/urandom on Unix-like systems or the RtlGenRandom system call on Windows. The instance does not automatically update its entropy pool, so it does not enjoy Yarrow’s key-compromise recovery properties.
