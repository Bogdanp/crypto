7 Randomness

Many cryptographic operations require a source of random numbers (or, equivalently, random bytestrings). In many environments, truly random numbers are impossible to obtain in sufficient quantity, so pseudo-random number generators designed for unpredictability and robustness to state compromise are used instead. A suitable source of pseudo-random numbers is called a cryptographically secure pseudo-random number generator (CSPRNG).

Note that ordinary pseudo-random number generators like Racket’s random function are not cryptographically secure. A CSPRNG that is improperly seeded (initialized), such as from a low-entropy source such as the current timestamp, is not cryptographically secure. See the notes on CSPRNG initialization in Cryptography Providers for more information.

procedure

(random-impl? v) → boolean?

  v : any/c
Returns #t if v is an implementation of a CSPRNG, #f otherwise.

procedure

(get-random [factories]) → random-impl?

  factories : (or/c crypto-factory? (listof crypto-factory?))
   = (crypto-factories)
Returns an implementation of a CSPRNG from factories, or #f if no factory in factories provides a CSPRNG implementation.

procedure

(random-bytes n [ri #:level level]) → bytes?

  n : exact-nonnegative-integer?
  ri : random-impl? = (get-random)
  level : (or/c 'strong 'very-strong) = 'strong
Generates n random bytes using the CSPRNG ri.

Some CSPRNGs may use the level argument to select the strength of the random numbers produced. The level 'strong is used to request material for use as nonces, random padding, IVs, and key material; the level 'very-strong is used to request long-term key material. However, a CSPRNG implementation may ignore the level argument, using the same process for all randomness requests.
